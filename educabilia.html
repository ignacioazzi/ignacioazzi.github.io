Introduction and objective of the project

As Educabilia was a small startup with only 5 members when I joined, my first role was as the UX/UI/visual designer. My job started at the whiteboard, then using Illustrator/Photoshop and finally Sublime Text. When the team grew and a more experienced UI developer joined, my role changed to solely UX/UI. Still, we worked closely together—sharing meetings, and pair programming.

My main objectives as a designer was:

  • Redesign the course details view, checkout flow, publish course flow.
  • Create the My Educabilia dashboard.
  • Design course landing pages.
  • Design the social media imagery for Facebook/Twitter.

Design process

We use the SCRUM framework, splitting our tasks into sprints. We had daily design meetings where we started sketching on a whiteboard for each task. This allowed us to work together, designing a consistent user flow.

When we reached a consensus, we moved on to design the wireframes in Illustrator. Once our own set of UI components were created, we usually sketched it out and then jumped directly to the browser.

We used semantic HTML5 for the markup, Sass + Compass for the visual styling, and as already mentioned, several small js libraries like Fayer, Bean,qwery among others. We also based our CSS architecture on concepts from SMACSS.
